    refactor(admin): 重构后台管理模块，优化鉴权体系与数据一致性
    
    本次提交对 Admin 模块进行了重构，涉及安全加固、性能优化及异步架构调整。
    
    1. 性能优化 (Performance)
       - 解决 N+1 问题 (用户列表): 在 AdminServiceImpl 中使用 MongoDB Aggregation 替代循环查询，一次性聚合统计帖数、粉丝数及获赞数据。
       - 解决 N+1 问题 (帖子列表): 使用 MyBatis-Plus in 查询批量补全用户邮箱信息。
       - 上下文优化: UserContext 新增 Role 缓存，结合 TokenInterceptor 实现一次解析多次复用，减少鉴权时的重复逻辑。
    
    2. 安全加固 (Security)
       - 垂直越权防护: 新增 AdminInterceptor，强制拦截非 ADMIN 角色的后台请求。
       - 鉴权体系升级: JWT Token Payload 新增 role 字段；TokenInterceptor 解析后自动注入 UserContext。
       - 并发安全: sendEmailCode 改用 Redis 原子操作 (setIfAbsent)，彻底杜绝并发验证码轰炸风险。
       - 参数规范化: 废弃 Map 弱类型参数，全量替换为 AdminUserDeleteDTO, AdminPostAuditDTO 等强类型对象。
       - 路由变更: 后台接口前缀由 /api/admin 迁移至 /admin，以便网关层做独立权限控制。
    
    3. 架构解耦与一致性 (Architecture)
       - 级联清理机制: 引入 UserDeleteEvent，管理员删除用户后异步清理 MongoDB 中的 7 类关联数据（帖子/评论/点赞/收藏/关注/历史等）。
       - 异步上下文透传: 改造 AsyncConfig，实现 MDC 和 UserContext 的跨线程透传，确保 @Async 任务中能获取当前用户信息。
       - 搜索解耦: 引入 UserSearchEvent，将搜索历史记录逻辑从 PostService 中剥离，避免主业务阻塞。
    
    4. 新增功能 (Features)
       - 用户详情聚合: 新增 getUserDetail 接口，聚合 PostgreSQL 基础信息与 MongoDB 行为数据（全维度影响力统计）。
       - 管理员功能: 补全 login, logout, send-code, reset-password, profile 等管理端专用接口。
    
    5. 测试 (Tests)
       - 集成测试: 新增 AdminControllerTest，覆盖登录鉴权、聚合查询、级联删除及内容审核场景。
       - 测试修正: 更新相关测试用例的 Token 生成逻辑，增加角色参数。
